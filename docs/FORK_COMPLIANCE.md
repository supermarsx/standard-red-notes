# Fork Compliance Notes

Standard Red Notes is a fork workspace based on AGPL-licensed upstream app and server code.

## Requirements

- Keep AGPL license files and copyright notices.
- Mark modified versions clearly.
- If the fork is served over a network, provide corresponding source to users as required by AGPL section 13.
- Do not use upstream trademarks or hosted service branding in a way that implies affiliation.
- Keep security contact and support links fork-owned before distributing binaries or hosted builds.

## Branding Policy

- User-visible product name: Standard Red Notes.
- Package import names can remain `@standardnotes/*` during the migration to avoid a high-risk rename.
- Public app IDs, bundle IDs, domains, and release artifact names should move to fork-owned identifiers before distribution.
